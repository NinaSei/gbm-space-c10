The principle

Why integrate?

Same cell type, different samples — should cluster by biology, not by batch
  • Multiple donors / samples / runs introduce technical differences = batch effects
  • A T cell from donor A and donor B are the same cell type — they should mix
  • Goal: remove technical variation, keep biological signal
  • Integration aligns shared cell populations across samples

Method 1

Harmony

Fast, CPU-friendly batch correction in PCA space
  • Operates on an existing PCA embedding (not raw counts)
  • Iterative loop: soft k-means clustering → estimate batch shifts → correct
  • Removes batch offsets within each cluster, keeps cluster structure
  • Linear-ish correction, no neural network to train
  • Fast, runs on CPU — a good default first pass
  • Returns a corrected embedding for neighbors / UMAP / clustering

Method 2

scVI

Deep generative model that learns a batch-free latent space
  • Variational autoencoder (VAE) — a neural network model
  • Models raw counts directly (negative-binomial), no pre-PCA
  • Conditions on batch → factors technical variation out of the latent space
  • Learns a low-dimensional latent embedding for downstream steps
  • GPU-accelerated; scales to very large datasets
  • Probabilistic → extras: denoising, normalized expression, DE
